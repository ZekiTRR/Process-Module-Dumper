# Process Module Dumper

Утилита на C++ для операционной системы Windows, предназначенная для перечисления загруженных модулей (DLL) и их базовых адресов в адресном пространстве процесса. Инструмент полезен для задач реверс-инжиниринга, отладки и анализа ПО.

## Основные функции

*   **Поиск по имени**: Находит процесс по его названию (например, `hl2.exe`).
*   **Извлечение PID**: Автоматически определяет идентификатор процесса.
*   **Перечисление модулей**: Выводит список всех загруженных библиотек и основного исполняемого файла.
*   **Базовые адреса**: Отображает начальные адреса модулей в памяти (VA).
*   **Экспорт данных**: Опциональное сохранение результатов в текстовый файл формата `<имя_процесса>_modules_dump.txt`.

## Как это работает

Программа использует Windows API (`tlhelp32.h`) и функцию `CreateToolhelp32Snapshot` с флагами `TH32CS_SNAPMODULE` и `TH32CS_SNAPMODULE32`. Это позволяет корректно получать данные как о 32-битных, так и о 64-битных модулях в целевом процессе.

## Требования

*   **ОС**: Windows (x86/x64).
*   **Компилятор**: Любой современный компилятор C++ с поддержкой Windows SDK (рекомендуется Visual Studio MSVC).
*   **Права доступа**: Для доступа к системным процессам или процессам, запущенным от имени другого пользователя, может потребоваться запуск программы от имени администратора.

## Инструкция по использованию

1.  Скомпилируйте файл `base_address.cpp`.
2.  Запустите скомпилированный файл.
3.  Введите имя процесса (включая расширение `.exe`).
4.  Подтвердите или отклоните сохранение лога (клавиши `y` или `n`).
